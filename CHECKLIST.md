# 動作確認チェックリスト

`tfx for Windows` の各機能を実機で確認するためのチェックリストです。
新機能を追加した後・リリース前にこのリストで一通り検証してください。

---

## 1. 起動とレイアウト

- [ ] `dotnet run` で起動するとウィンドウタイトルが `tfx 0.4.4 - <パス>` になる
- [ ] OS の表示言語が日本語の場合、メニュー、ツールチップ、ステータス、プレビュー情報が日本語になる
- [ ] OS の表示言語が日本語以外の場合、メニュー、ツールチップ、ステータス、プレビュー情報が英語になる
- [ ] サイドバー（PINNED / FOLDERS）、左ペイン、右ペイン、プレビューペインが正しく配置される
- [ ] 既定ではスプリット表示（左右ペイン）かつプレビュー表示
- [ ] アクティブなペインの枠が明るい緑、非アクティブはグレー
- [ ] ステータスバー左に件数、右にドライブの空き容量（例: `C:\  120 GB free of 476 GB`）と現在のバージョンが表示される
- [ ] ウィンドウ幅／高さは前回終了時のサイズで開く

## 2. ツールバー

- [ ] ボタンがアイコン（テキストではなく）で表示される
- [ ] Navigation / Path-Search / View / Utility の順に並ぶ
- [ ] 各ボタンにマウスオーバーするとツールチップとショートカットが出る
- [ ] ボタン hover で背景色が変化、Preview / Split / Hidden の ON 状態は青背景になる
- [ ] 各ボタンを押すと該当機能が動作する

## 3. ナビゲーション

- [ ] フォルダツリーをワンクリック → アクティブペインに内容が表示
- [ ] `▶`（展開三角）クリックは展開のみで、ナビゲートしない
- [ ] パンくずバーの各セグメントクリックで該当階層へ
- [ ] パンくずバーに長いパスを表示したとき、右端のフォルダ名が見え、左側が見切れる
- [ ] パンくずバー右側の空き部分クリックで編集モード
- [ ] ドライブ直下に達したら ChevronUp / Backspace は無反応（親なし）
- [ ] 行ダブルクリックでフォルダに入る／ファイルが既定アプリで開く
- [ ] `..` 行はクリックで選択のみ、ダブルクリックで親フォルダへ移動し、元いたフォルダ名にフォーカスが戻る
- [ ] `..` 行を選択して Enter で親フォルダへ
- [ ] フォルダ移動直後は `..` のファイル名セルにフォーカスがあり、上下キーで一覧選択が移動する
- [ ] `..` または Backspace で親フォルダへ戻ると、元いたフォルダ名のセルにフォーカスがある
- [ ] 空フォルダで選択がない状態でも、上下キーで `..` を選択できる
- [ ] Backspace で親フォルダへ（テキスト入力中は通常通り文字消去）
- [ ] `Ctrl + [` で戻る、`Ctrl + ]` で進む
- [ ] `Ctrl + Up` で親フォルダ
- [ ] Tab → FolderTree → 左ペイン → 右ペイン → FolderTree の循環
- [ ] Shift + Tab で逆順
- [ ] Left / Right キーで左右ペインのフォーカス切替

## 4. アドレスバー

- [ ] `Ctrl + L` で編集モード
- [ ] `F4` で編集モード
- [ ] 上部の現在パスをクリックするとアクティブペインのパス入力欄になる
- [ ] パンくずバーをダブルクリックするとパス入力欄になる
- [ ] 編集モード中の Enter で入力したパスへ移動
- [ ] `%USERPROFILE%`, `%TEMP%` 等の環境変数が展開される
- [ ] 不正なパスではステータスバーに `Folder not found:` が出る
- [ ] Esc で編集モードをキャンセル

## 5. 選択

- [ ] 行クリックで単独選択
- [ ] Ctrl + クリックで個別追加 / 解除
- [ ] Shift + クリックで範囲選択
- [ ] ファイル一覧の空白からドラッグして複数選択できる
- [ ] 複数選択済みの項目上からドラッグすると、選択を維持したままドラッグ＆ドロップできる
- [ ] `Ctrl + A` で全選択
- [ ] 選択するとステータスバーが `K of N selected (合計サイズ)` に更新
- [ ] 親フォルダ行（`..`）はサイズ集計から除外される
- [ ] 詳細表示の Size 列の数値は右詰めで表示される
- [ ] 詳細表示の各テキスト列は、右側に約 1 文字分の余白がある

## 6. ファイル操作

- [ ] `Ctrl + N` または右クリック New Folder → フォルダ名入力ダイアログ → 作成
- [ ] `Ctrl + Shift + N` または右クリック New File → ファイル名入力ダイアログ → 空ファイル作成
- [ ] 同名フォルダがあると `(2)`, `(3)` の連番が付く
- [ ] `F2` でインラインリネーム。拡張子部分が選択されない（フォルダなら全選択）
- [ ] Enter で確定、Esc でキャンセル
- [ ] `Ctrl + C` → `Ctrl + V` でコピー
- [ ] `Ctrl + X` → `Ctrl + V` で移動
- [ ] 同名がある時、貼り付けで `(2)` 等の連番付与
- [ ] OS のクリップボード経由でも同じ動作（外部からのコピーがペースト可能）
- [ ] `Delete` でごみ箱へ（確認ダイアログあり）
- [ ] `Shift + Delete` で完全削除（警告メッセージ付き確認ダイアログ）
- [ ] 削除確認ダイアログがアプリのダークテーマ色で表示される
- [ ] `Ctrl + K` または右クリック Compress to Zip → 選択項目から zip 作成
- [ ] `.zip` を選択して `Ctrl + Shift + E` または右クリック Extract Zip → 同名フォルダへ展開
- [ ] zip 作成／展開先が既存名と衝突する場合 `(2)`, `(3)` の連番が付く

## 7. ドラッグ＆ドロップ（修飾キー）

- [ ] **通常ドラッグ**: 同一ドライブ → 移動、別ドライブ → コピー
- [ ] 同じフォルダ内へ通常ドラッグしても `(2)` 付きの重複名が作成されない
- [ ] **Shift + ドラッグ**: カーソルが「移動」、強制移動
- [ ] **Ctrl + ドラッグ**: カーソルが「+ コピー」、強制コピー
- [ ] **Alt + ドラッグ**: カーソルが「→ ショートカット」、ドロップ先に `<元名> - Shortcut.lnk` 作成
- [ ] tfx → エクスプローラへドラッグ後、tfx 側のリスト表示が更新される
- [ ] エクスプローラ → tfx へドロップで規則どおり動作
- [ ] ファイル一覧内のフォルダ行 / フォルダアイコンへドロップすると、そのフォルダ内へ移動またはコピーされる
- [ ] 別ドライブへのフォルダドロップが成功する（cross-volume）

## 8. 表示モード

- [ ] ツールバーの ViewMode ボタンで Details ⇔ Icons を切替
- [ ] **Details**: 列ヘッダクリックでソート、▲▼ 表示
- [ ] **Icons**: 32×32 アイコン + ファイル名のグリッド
- [ ] Icons モードで選択・ダブルクリック・右クリック・ドラッグ＆ドロップが動作
- [ ] Icons モードで `Ctrl + A` 全選択
- [ ] モード切替が次回起動時にも維持される

## 9. 列管理（Details モード）

- [ ] Columns ボタンで Popup が開く
- [ ] チェックボックスで表示／非表示切替（Popup は開いたまま）
- [ ] ▲ / ▼ で順序変更（左右ペイン両方に反映）
- [ ] 最後の 1 列を非表示にしようとすると警告
- [ ] 順序と表示状態が次回起動時に維持される

## 10. ソート

- [ ] Name / Date modified / Type / Size / Date created / Owner / Attribute すべてでソート可能
- [ ] Name 列のヘッダー境界をマウスでドラッグすると、ファイル名列の幅を変更できる
- [ ] Name 以外の列は、ヘッダー境界をドラッグしても幅変更されない
- [ ] 同じ列をクリックすると Asc/Desc がトグル
- [ ] 並び替え後も親フォルダ（`..`）は常に先頭
- [ ] フォルダがファイルより先に表示される
- [ ] 別フォルダにナビゲートしても直前のソートが維持される

## 11. 検索

- [ ] `Ctrl + F` で SearchBox にフォーカス
- [ ] 部分一致でリアルタイム絞り込み
- [ ] 親フォルダ（`..`）はフィルタを受けず常に表示
- [ ] Esc で検索クリアしてフォーカス復帰

## 12. プレビュー

- [ ] Preview ボタンで開閉
- [ ] 閉じるとファイル一覧が右いっぱいまで広がる
- [ ] スプリット表示時、閉じても左右ペインの幅比率が保たれる
- [ ] テキストファイル（.txt, .md, .json, .cs ... ）が読める範囲で表示
- [ ] EUC-JP / ISO-2022-JP (JIS) / Shift_JIS のテキストが文字化けせず表示される
- [ ] UTF-8 は BOM あり / BOM なしが Encoding に表示される
- [ ] テキストプレビューに Encoding と Newline が表示される
- [ ] 画像ファイル（.png, .jpg, .gif, .bmp ... ）が表示
- [ ] PDF ファイルは `pdftoppm` または Windows のサムネイルプロバイダーで先頭ページのプレビューが表示される
- [ ] PDF レンダラーがない環境では、異常終了せず情報表示のみになる
- [ ] バイナリファイルは情報のみ
- [ ] フォルダや `..` 行はメタデータのみ表示

## 13. ピン留め

- [ ] 初回起動時に既定のピン（ホーム / Desktop / Documents / Downloads）
- [ ] ツールバーの Pin ボタンで現在のフォルダをピン留め
- [ ] ピン留めされたフォルダ名の右クリックメニューからピン留めを外せる
- [ ] ピンをドラッグで並び替え
- [ ] ピンをワンクリックでアクティブペインに移動
- [ ] 20 文字以上のピン表示は `C: ... \Downloads` のように中央省略され、ツールチップでフルパスが見える
- [ ] 順序が次回起動時に維持される

## 14. 右クリックコンテキストメニュー

- [ ] 行を右クリックすると当該行が選択されてメニュー表示
- [ ] 既選択行の右クリックでは選択を維持
- [ ] Open / Reveal in Explorer
- [ ] Cut / Copy / Paste（Paste はクリップボードの状態で有効/無効）
- [ ] Compress to Zip / Extract Zip（Extract は zip 選択時のみ有効）
- [ ] Rename / Move to Recycle Bin / Delete permanently
- [ ] New Folder / New File / Open Terminal here
- [ ] Copy current path で選択中のファイルまたはフォルダのフルパスがクリップボードに入る

## 15. 外部連携

- [ ] Terminal ボタン / `Ctrl + Shift + T` で現在のフォルダをカレントとして PowerShell（または Windows Terminal）が起動
- [ ] Explorer ボタンで現在パスがエクスプローラで開く（選択中ファイルがあれば選択状態）
- [ ] `Ctrl + Shift + .` または Hidden ボタンで隠しファイル表示切替
- [ ] Hidden OFF では Windows Hidden 属性の項目と `.git` / `.env` など `.` 始まりの項目が非表示になる
- [ ] Hidden OFF ではフォルダツリーにも Hidden 属性フォルダと `.` 始まりのフォルダが表示されない
- [ ] Hidden ON にするとフォルダツリーにも Hidden 属性フォルダと `.` 始まりのフォルダが表示される

`%APPDATA%\tfx\settings.json` を確認：

- [ ] LeftPath / RightPath
- [ ] ActivePane
- [ ] ShowSplit / ShowPreview / ShowHidden
- [ ] Left / Top / Width / Height / IsMaximized
- [ ] SidebarWidth / PreviewWidth / LeftPaneRatio
- [ ] PinnedFolders（順序）
- [ ] VisibleFileColumns
- [ ] FileColumnOrder
- [ ] ViewMode（`Details` / `Icons`）
- [ ] 再起動すると前回終了時のパス、アクティブペイン、表示モード、列、サイドバー幅、プレビュー幅、左右ペイン比率、ウィンドウ位置が復元される
- [ ] ファイルを削除して再起動 → 既定状態に戻る

## 17. 起動時引数

```powershell
dotnet run -- "C:\path\to\folder"
```

- [ ] 引数で指定したフォルダが左ペインに表示される
- [ ] 引数なしでは前回終了時のパスを復元
- [ ] 存在しないパスを指定すると、前回パス（または UserProfile）にフォールバック
